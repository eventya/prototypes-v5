Prototype · WhatsApp Onboarding — 8. UI ↔ Senders API
← înapoi la set
Ce se automatizează, pas cu pas
Fiecare acțiune din panou → un apel Twilio Senders API
Automat, prin platformă Rulează în fundal (webhook) Așteptare la Meta (nu o controlezi)
1
Client trimite formularul + alege SMS/Voce
Telefon, display name, poză, metoda de verificare. La create, Meta trimite automat OTP-ul pe număr.
POST /v2/Channels/Senders
{ sender_id:"whatsapp:+40…", profile.name, profile.logo_url,
  configuration.verification_method:"voice"|sms, webhook.callback_url }
→ Status: CREATING · OTP-ul pleacă automat pe număr
2
Codul ajunge pe telefonul clientului
SMS sau apel vocal, în funcție de verification_method. Nu e un apel API separat — îl trimite Meta.
(fără apel din partea ta — clientul citește codul de pe telefonul lui)
3
Clientul introduce codul de 6 cifre → „Verifică"
Îl tastează el, în panou — nimeni nu-l sună.
POST /v2/Channels/Senders/{Sid} — { configuration.verification_code:"481234" }
→ Sender status: VERIFYING
4
În fundal: urmărești statusul sender-ului
Prin webhook-ul dat la create (Twilio → tine) sau polling. Panoul se actualizează singur.
GET /v2/Channels/Senders/{Sid} — sau callback pe webhook.callback_url
→ CREATING → VERIFYING → OFFLINE (tranzitoriu) → ONLINE
5
Meta aprobă numele afișat
Singurul pas pe care nu-l controlezi. Ore–zile, asincron. Nu se rezolvă cu un telefon oricum.
(fără apel din partea ta — aștepți callback-ul) → status ONLINE
→ Panoul devine automat „Activ". Dacă numele e respins → limită 250 msg/24h; clientul editează numele → POST /v2/Channels/Senders/{Sid} (update profile.name) → re-review.
Pe scurt
  • Pașii 1–4 sunt 100% prin platformă: fără consolă Twilio, fără apel la client.
  • Pasul 5 (aprobare Meta) e o așteptare, nu o muncă manuală — devine o stare în panou.
  • Endpoint-urile sunt cele reale din Senders API - WhatsApp (twilio.com/docs/whatsapp/register-senders-using-api, GA). Twilio recomandă API-ul pentru volum (bulk); pentru puține numere → Self Sign-up. Numere fixe = verification_method:"voice"; IVR/robot nu pot primi OTP.
